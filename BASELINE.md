# 재베이스라인 기록 (게이트 3)

업그레이드(fp16 자동 + Muon + 토큰예산 1B + loop off + compile + 프리페치)의
효과를, 같은 하드웨어에서 base 사전학습 **1회 완주** 전후로 비교해 기록한다.
목표: **총 시간이 이전 대비 한 자릿수 배 단축**.

> 이 표는 Kaggle T4에서 실제 학습을 돌린 뒤 채운다(코드는 준비 완료, 수치는
> 런타임 산출물). 관측 tok/s로 예산 시간을 먼저 추정하고 실측으로 갱신한다.

## 설정

| 항목 | 이전(full-loop) | 이후(full, base) |
|---|---|---|
| loop | 2..6 ×2 (연산 1.5배) | off |
| compile | off (확률적 그래프) | on |
| AMP dtype | bfloat16(T4 가속 없음) | float16(자동 선택) |
| 옵티마이저 | AdamW | Muon+AdamW |
| 토큰 예산 | ~9.8B (40k스텝) | 1.0B (~4069스텝) |
| 유효 배치 | 480×512 | 480×512 |
| 데이터로더 | 동기 | 프리페치(더블버퍼) |

## 수치 (채울 것)

| 지표 | 이전 | 이후 | 배수 |
|---|---|---|---|
| ms/step | — | — | — |
| 최종 val loss | — | — | — |
| 총 소요시간 | — | — | — |

## 예산 시간 추정식

```
예상 시간 = target_tokens ÷ (유효배치토큰) × (ms/step) 
         = max_steps × ms/step
```

`pretrain.py`가 시작 시 `토큰 예산 ... -> N steps`를 출력하고, `AMP dtype: ...`,
`Muon 대상 .. / AdamW 대상 ..`을 로그로 남긴다. 첫 100스텝의 ms/step으로 총
시간을 추정한 뒤 실측으로 이 표를 갱신한다.
